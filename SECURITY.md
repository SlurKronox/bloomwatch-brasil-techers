# Security Policy

## Supported Versions
Security updates are applied to the `main` branch.

| Version | Supported |
| --- | --- |
| main | Yes |
| other branches | No |

## Reporting a Vulnerability
Use GitHub Security Advisories ("Report a vulnerability") when available.
If unavailable, open an issue with prefix `[SECURITY]` and avoid publishing exploit details.

Please include:
- Affected component and impact
- Reproduction path
- Mitigation suggestion

## Response Targets
- Initial response: up to 72 hours
- Mitigation plan: up to 7 business days
- Public disclosure after fix release

## Scope
This policy covers frontend code, Supabase integration files, CI workflows, and repository metadata.
